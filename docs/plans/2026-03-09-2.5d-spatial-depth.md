# 2.5D Spatial Depth Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Y座標ベースのエンティティスケール変化とグリッド間隔勾配で、2Dゲーム画面に奥行き感を追加する。

**Architecture:** SyncRenderSystem で各エンティティの Y 座標から `depthScale`（0.75〜1.0）を計算し、RenderEntity に渡す。GameCanvas の EntitySlot が depthScale を使って描画サイズを調整。グリッドの水平線間隔も Y に応じて可変にする。衝突判定は一切変更しない。

**Tech Stack:** @shopify/react-native-skia, react-native-reanimated SharedValue, TypeScript

function computeDepthScale(y: number, visibleHeight: number): number {
  const ratio = Math.max(0, Math.min(1, y / visibleHeight));
  return DEPTH_SCALE_MIN + (1 - DEPTH_SCALE_MIN) * ratio;
}

### Task 5: Implement grid spacing gradient

**Files:**
- Modify: `src/rendering/GameCanvas.tsx`
- Reference: `src/constants/balance.ts` (GRID_PITCH_MIN, GRID_PITCH_MAX)

**Context:** Currently, `GameCanvasInner` renders horizontal grid lines using `GridHLine` components with equal spacing (`GRID_H_SPACING = 80 * scale`). Each `GridHLine` uses a `useDerivedValue` to compute its Y position based on scroll offset and fixed index × spacing.

The grid gradient requires variable spacing (dense at top, sparse at bottom). Since each `GridHLine` is a React component with its own `useDerivedValue`, we need to change the Y computation to account for non-uniform spacing. The approach: pre-compute each line's **base offset** using accumulated variable pitch, then use that offset in the scroll calculation.

**Step 1: Import the new constants**

Add `GRID_PITCH_MIN` and `GRID_PITCH_MAX` to the import from `@/constants/balance` (line 18).

**Step 2: Replace the GridHLine component**

Replace the existing `GridHLine` component (lines 292-316) with a version that accepts a `baseOffset` prop instead of computing position from `index * spacing`:

```typescript
function GridHLine({
  baseOffset,
  totalCycleHeight,
  scrollY,
  scale,
  width,
  height,
}: {
  baseOffset: number;
  totalCycleHeight: number;
  scrollY: GameCanvasProps['scrollY'];
  scale: number;
  width: number;
  height: number;
}) {
  const y = useDerivedValue(
    () => ((scrollY.value * scale + baseOffset) % totalCycleHeight) - totalCycleHeight * 0.1
  );
  const gridOpacity = useDerivedValue(() => {
    const ratio = y.value / height;
    return 0.15 + Math.max(0, Math.min(1, ratio)) * 0.4;
  });
  return <Rect x={0} y={y} width={width} height={1} color="#2a2a4e" opacity={gridOpacity} />;
}
```

**Step 3: Update GameCanvasInner to compute variable grid offsets**

Replace the current grid horizontal setup (lines 351-363) and the gridHSlots JSX (lines 375-378) with:

```typescript
  // Grid lines — horizontal lines with perspective gradient spacing
  const GRID_BASE_SPACING = 80 * scale;
  const GRID_V_SPACING = 80 * scale;
  const GRID_V_COUNT = Math.ceil(width / GRID_V_SPACING) + 1;

  // Pre-compute grid line offsets with variable spacing (dense at top → sparse at bottom)
  const gridHLines = React.useMemo(() => {
    const lines: { baseOffset: number }[] = [];
    let y = 0;
    const totalHeight = height * 1.2; // extra buffer for scroll wrapping
    while (y < totalHeight) {
      lines.push({ baseOffset: y });
      const ratio = y / totalHeight;
      const pitchMultiplier = GRID_PITCH_MIN + (GRID_PITCH_MAX - GRID_PITCH_MIN) * ratio;
      y += GRID_BASE_SPACING * pitchMultiplier;
    }
    return lines;
  }, [height, scale]);

  const totalCycleHeight = React.useMemo(() => {
    if (gridHLines.length === 0) return height;
    return gridHLines[gridHLines.length - 1].baseOffset + GRID_BASE_SPACING * GRID_PITCH_MAX;
  }, [gridHLines, height, scale]);
```

Then in the JSX, replace the horizontal grid section:

```tsx

      {gridHLines.map((line, i) => (
        <GridHLine
          key={`gh${i}`}
          baseOffset={line.baseOffset}
          totalCycleHeight={totalCycleHeight}
          scrollY={scrollY}
          scale={scale}
          width={width}
          height={height}
        />
      ))}
```

Remove the now-unused `GRID_H_SPACING`, `GRID_H_COUNT`, and `gridHSlots` variables.
