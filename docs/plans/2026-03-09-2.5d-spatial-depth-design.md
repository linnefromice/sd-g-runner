# 2.5D Spatial Depth Design

**Goal:** Y座標ベースのスケール変化とグリッド間隔勾配で、2Dゲーム画面に奥行き感を追加する。

**Architecture:** SyncRenderSystem で depthScale を計算し RenderEntity 経由で Skia 描画に渡す。衝突判定（論理座標）は一切変更しない。グリッドは水平線の間隔を Y 位置に応じて可変にする。

**Tech Stack:** @shopify/react-native-skia, react-native-reanimated SharedValue

---

## 1. Y-Based Entity Depth Scale

エンティティの Y 座標に基づき、画面上部（遠く）では小さく、下部（近く）では元サイズで描画。

### スケール計算

```
depthScale = DEPTH_SCALE_MIN + (1 - DEPTH_SCALE_MIN) × clamp(y / visibleHeight, 0, 1)
```

- `DEPTH_SCALE_MIN = 0.75` (画面最上端で 75% サイズ)
- Y=0 → 0.75x, Y=visibleHeight → 1.0x
- Y < 0（画面外上部）→ 0.75x にクランプ

### 適用対象

| エンティティ | depthScale 適用 | 理由 |
|-------------|----------------|------|
| enemies | Yes | 上から降ってくる → 遠→近の効果が顕著 |
| debris | Yes | 同上 |
| gates | Yes | 同上 |
| boss | No | 画面上部に固定 → スケール変化が不自然 |
| player | No | プレイヤー操作対象 → サイズ固定が直感的 |
| playerBullet | No | 上方向に飛ぶ → 小さくなると弱そうに見える |
| enemyBullet | Yes | 上から降ってくる → 自然な効果 |
| particles | No | 小さすぎてスケール効果が見えない |
| shockwave/exBeam/laser | No | エフェクト系はスケール不要 |

### 衝突判定

変更なし。論理座標はそのまま、視覚のみスケール。

## 2. Grid Spacing Gradient

背景グリッドの水平線間隔を Y 位置に応じて変化させる。上部（遠く）は密、下部（近く）は疎。

### 間隔計算

```
currentPitch = basePitch × (GRID_PITCH_MIN + (GRID_PITCH_MAX - GRID_PITCH_MIN) × (y / height))
```

- `GRID_PITCH_MIN = 0.6` (上端: base pitch × 0.6 → 密)
- `GRID_PITCH_MAX = 1.4` (下端: base pitch × 1.4 → 疎)
- 水平線のみ可変（垂直線は等間隔のまま）

### 描画方法

等間隔ループから累積ループに変更:

```
y = startOffset
while (y < height) {
  draw horizontal line at y
  ratio = y / height
  currentPitch = basePitch × (GRID_PITCH_MIN + (GRID_PITCH_MAX - GRID_PITCH_MIN) × ratio)
  y += currentPitch
}
```

既存の Y-opacity (`0.15 + y/height × 0.4`) はそのまま維持。

## パフォーマンス影響

- depthScale 計算: entity あたり乗算 2 回 → 50 entity で ~100 ops（無視可能）
- path 再計算: depthScale を buildPath の w/h に掛けるだけ（既存処理の延長）
- グリッド: ループ内乗算 1 回追加（~30 回/frame）
- 新規 useDerivedValue: 0 個
